### Formatter Implementation (`output/formatter.go`)
```go
// Render outputs data in the configured format
func (f *Formatter) Render(headers []string, rows []TableRow, data interface{}) error {
	switch f.Format {
	case "json":
		return f.JSON(data)
	case "yaml":
		return f.YAML(data)
	default:
		return f.Table(headers, rows)
	}
}
```

## 6. Constraints and Gotchas
- **Error Wrapping**: Always use `fmt.Errorf("context: %w", err)`.
- **IO Streams**: Use `cmd.OutOrStdout()` and `cmd.ErrOrStderr()` to ensure testability.
- **Imports**: Use project-relative paths: `github.com/kdobmayer/kond-test-go-cli/...`.
- **Duplicated Logic**: The current `cmd/status.go` contains "intentional rough edges" with manual formatting. These should be refactored to use `output.Formatter`.
- **Flag Precedence**: The `--json` flag on the `status` command should take precedence over the global `-o/--output` flag if both are provided, or simply set the output format to JSON.
