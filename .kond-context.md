# Context Bundle: Add --no-color flag

## Goal
Implement a `--no-color` flag to disable colored output. Since the project currently does not use colored output, the implementer will likely need to introduce a color library (e.g., `github.com/fatih/color`) and add colored status indicators (e.g., Green for "completed", Red for "failed") while ensuring the flag can disable them.

## Test Command
`go test ./...`

## Build Command
`go build -o pipeline .`

## Relevant Code Snippets

### Persistent Flag Registration (`cmd/root.go`)
Existing pattern for persistent flags:
```go
var (
	outputFormat string
)

func init() {
	rootCmd.PersistentFlags().StringVarP(&outputFormat, "output", "o", "table", "Output format: table, json, yaml")
}
```

## Test Patterns (`cmd/cmd_test.go`)
Tests use `rootCmd` and capture output via `SetOut`/`SetErr`:
```go
func TestSomeCommand(t *testing.T) {
	dir := t.TempDir()
	rootCmd.SetArgs([]string{"cmd", "arg"})
	var buf bytes.Buffer
	rootCmd.SetOut(&buf)
	rootCmd.SetErr(&buf)

	if err := rootCmd.Execute(); err != nil {
		t.Fatalf("command error = %v", err)
	}
    // Assert on buf.String()
}
```

## Files to Modify/Create
- `cmd/root.go`: Add the `--no-color` persistent flag.
- `output/formatter.go`: Update `Formatter` to handle color state.
- `cmd/run.go`: Add colors to execution results table.
- `cmd/status.go`: Add colors to status tables/output.
- `config/config.go`: (Optional) Add `NoColor` to the persistent configuration.

## Constraints and Gotchas
- **No existing color library**: You will need to add one to `go.mod`.
- **Cobra Flags**: Use `rootCmd.PersistentFlags()` so it's available globally.
- **SilenceUsage**: The project doesn't explicitly set `SilenceUsage`, but error handling is centralized in `rootCmd.Execute()`.
- **Duplication**: Be aware of the "intentional rough edges" where output formatting is duplicated in `cmd/status.go` and `cmd/run.go`. Consider refactoring to use `output.Formatter` or ensuring color is handled in both places.
