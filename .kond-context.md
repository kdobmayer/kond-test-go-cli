### cmd/cmd_test.go (Test Structure, Output Capture, Error Assertion)

These snippets demonstrate how to set up tests for Cobra commands, capture output, and assert on both successful execution and expected errors.

```go
func TestInitCommand(t *testing.T) {
	dir := t.TempDir()

	rootCmd.SetArgs([]string{"init", "test-pipeline", "-d", dir})
	var buf bytes.Buffer
	rootCmd.SetOut(&buf)

	if err := rootCmd.Execute(); err != nil {
		t.Fatalf("init command error = %v", err)
	}
	// ... assert output ...
	if !bytes.Contains(buf.Bytes(), []byte("expected string")) {
		t.Error("expected 'expected string' in output")
	}
}

func TestInitCommand_AlreadyExists(t *testing.T) {
	dir := t.TempDir()
	path := filepath.Join(dir, "existing.yaml")
	os.WriteFile(path, []byte("name: existing"), 0644)

	rootCmd.SetArgs([]string{"init", "existing", "-d", dir})
	var buf bytes.Buffer
	rootCmd.SetOut(&buf)
	rootCmd.SetErr(&buf)

	err := rootCmd.Execute()
	if err == nil {
		t.Error("expected error when file already exists")
	}

}
```

## 6. Constraints and Gotchas

*   **Error Handling:**
    *   Errors MUST be wrapped with `fmt.Errorf("context: %w", err)` to preserve the chain.
    *   Functions return `(Type, error)` tuples; `nil` checks are required.
    *   `cobra.Command RunE` functions MUST return an error (or `nil`).
    *   `main.go` is the ONLY place to call `os.Exit(1)`.
*   **Output Conventions:**
    *   All command output (stdout/stderr) MUST use `cmd.OutOrStdout()` and `cmd.ErrOrStderr()`.
    *   Direct `fmt.Println` or `os.Stdout/os.Stderr` are only for `main.go` or `output` package functions.
    *   For simple output like printing "bar", `fmt.Fprintln(cmd.OutOrStdout(), "bar")` is appropriate. For structured output, `output.NewFormatter` should be used.
*   **Cobra Command Structure:**
    *   New subcommands register in their `init()` functions using `rootCmd.AddCommand(myCommand)`.
    *   Global flags should use `PersistentFlags()` on `rootCmd`.
    *   Command logic resides in the `RunE` function.
*   **Testing Conventions:**
    *   Tests are co-located in `_test.go` files (e.g., `cmd/root_test.go`).
    *   Use `t.TempDir()` for temporary directories in tests.
    *   Simulate CLI using `rootCmd.SetArgs()`, `SetOut()`, `SetErr()` with `bytes.Buffer`.
    *   `t.Cleanup()` can be used for test state reset.
*   **"Rough Edge" Warning:** Be mindful that `cmd/run.go` and `cmd/logs.go` currently have some duplicated output formatting logic (e.g., `json.NewEncoder`, `yaml.NewEncoder`) directly in their `RunE` functions instead of fully delegating to `output.Formatter`. For the `--bar` flag, simple `fmt.Fprintln` to `cmd.OutOrStdout()` is sufficient, but be aware of this pattern for more complex output.
