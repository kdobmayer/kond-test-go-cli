# Context Bundle for 'tail' subcommand

## Goal
Add a 'tail' subcommand that reads from stdin and prints the last N lines (default 10). Support a -n flag to specify the number of lines.

## 1. Test Command
```bash
go test -race -cover ./...
```

## 2. Build Command
```bash

```

## 3. Relevant Code Snippets

### Command Registration (from `cmd/root.go` and `cmd/run.go`)
This shows how commands are defined and added to the `rootCmd`.

```go
// cmd/root.go
package cmd

import (
	"github.com/spf13/cobra"
)

var (
	outputFormat string
)

var rootCmd = &cobra.Command{
	Use:   "pipeline",
	Short: "A CLI tool for managing and executing pipelines",
	Long: `Pipeline is a CLI tool that allows you to define, execute, and monitor
multi-step pipelines with dependency management and parallel execution.`,
}

func init() {
	rootCmd.PersistentFlags().StringVarP(&outputFormat, "output", "o", "table", "Output format: table, json, yaml")
}

func Execute() error {
	return rootCmd.Execute()
}

// cmd/run.go
package cmd

import (
	"fmt"
	"os"

	"github.com/kdobmayer/kond-test-go-cli/config"
	"github.com/kdobmayer/kond-test-go-cli/output"
	"github.com/kdobmayer/kond-test-go-cli/pipeline"
	"github.com/spf13/cobra"
)

var runCmd = &cobra.Command{
	Use:   "run [pipeline-file]",
	Short: "Execute a pipeline",
	Long:  `Run a pipeline defined in a YAML file, executing steps in dependency order.`,
	Args:  cobra.ExactArgs(1),
	RunE:  runPipeline,
}

func init() {
	rootCmd.AddCommand(runCmd)
	runCmd.Flags().Bool("dry-run", false, "Show execution plan without running")
	runCmd.Flags().Bool("verbose", false, "Show step output in real-time")
}
```

### Error Handling (from `main.go` and `cmd/run.go`)
Errors are wrapped with context and returned. `main.go` handles the final exit.

```go

	"fmt"
	"os"

	"github.com/kdobmayer/kond-test-go-cli/cmd"
)

func main() {
	if err := cmd.Execute(); err != nil {
		fmt.Fprintf(os.Stderr, "Error: %v

// cmd/run.go (example of error wrapping)
```

### Test Patterns (from `cmd/cmd_test.go`)
How tests are structured using `t.TempDir()`, `rootCmd.SetArgs()`, `rootCmd.SetOut()/SetErr()`, `rootCmd.Execute()`, and `bytes.Contains()`.

```go
// cmd/cmd_test.go (example)
package cmd

import (
	"bytes"
	"os"
	"path/filepath"
	"testing"
)

func TestInitCommand(t *testing.T) {
	dir := t.TempDir()

	rootCmd.SetArgs([]string{"init", "test-pipeline", "-d", dir})
	var buf bytes.Buffer
	rootCmd.SetOut(&buf)

	if err := rootCmd.Execute(); err != nil {
		t.Fatalf("init command error = %v", err)
	}

	// Verify file was created
	path := filepath.Join(dir, "test-pipeline.yaml")
	if _, err := os.Stat(path); err != nil {
		t.Errorf("pipeline file not created: %v", err)
	}
}
```

## 5. Constraints
*   Errors are always wrapped using `fmt.Errorf("context: %w", err)`.
*   Functions consistently return `(Type, error)` tuples. Perform nil checks on errors.
*   Command logic in Cobra `RunE` functions must return an `error`.
*   Only `main.go` should call `os.Exit(1)`.
*   Use `cmd.OutOrStdout()` and `cmd.ErrOrStderr()` for all command output.
*   Adhere to `outputFormat` if structured output were ever needed (unlikely for `tail`).
