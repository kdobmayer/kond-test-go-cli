This bundle provides the necessary context for an implementation agent to add a `--foo` flag to the CLI. The flag should print 'foo' to stdout and exit with a success status (0).

## 3. Relevant Code Snippets

### `main.go` - Top-level Error Handling

This snippet shows how the root command is executed and how top-level errors are handled, leading to `os.Exit(1)`. Successful execution will result in `os.Exit(0)` implicitly by the shell. The `RunE` function should *not* call `os.Exit()` directly.

```go
// main.go
func main() {
    if err := cmd.Execute(); err != nil {
        fmt.Fprintf(os.Stderr, "Error: %v
", err)
        os.Exit(1)
    }
}
```

### `cmd/root.go` - `rootCmd` Definition and Persistent Flag Registration

This snippet demonstrates the definition of the `rootCmd` and how persistent flags are registered. The new `--foo` flag should be added here.

```go
// cmd/root.go
package cmd

import (
	"github.com/spf13/cobra"
)

var (
	outputFormat string
)

var rootCmd = &cobra.Command{
	Use:   "pipeline",
	Short: "A CLI tool for managing and executing pipelines",
	Long: `Pipeline is a CLI tool that allows you to define, execute, and monitor
multi-step pipelines with dependency management and parallel execution.`,
}

func init() {
	rootCmd.PersistentFlags().StringVarP(&outputFormat, "output", "o", "table", "Output format: table, json, yaml")
}

func Execute() error {
	return rootCmd.Execute()
}
```

## 6. Constraints and Gotchas

*   **Error Handling:** Cobra `RunE` functions MUST return an `error`. The `main.go` function is responsible for calling `os.Exit(1)` if `cmd.Execute()` returns an error. The `RunE` for `--foo` should return `nil` for success, not call `os.Exit(0)`.
*   **CLI Output:** All CLI output must go through `cmd.OutOrStdout()` or `cmd.ErrOrStderr()`. For the `--foo` flag, direct `fmt.Fprint(cmd.OutOrStdout(), "foo
")` is appropriate.
*   **Cobra Configuration:** Consider setting `rootCmd.SilenceUsage = true` and `rootCmd.SilenceErrors = true` within the `main` function or the `init` function of `root.go` to prevent Cobra from printing usage or error messages automatically, as the `main` function handles error reporting. This ensures clean output for simple commands like `--foo`.
*   **Flag Persistence:** The `--foo` flag should likely be a *persistent* flag on `rootCmd` if it's intended to be available globally before any subcommand.
*   **`output.Formatter` Usage:** For this simple task (printing "foo"), the `output.Formatter` package is not necessary. Direct writing to `cmd.OutOrStdout()` is sufficient and aligns with the "Output Formatter Exception" noted in the project context.
