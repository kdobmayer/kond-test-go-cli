```go
// executeLevel runs all steps in a level concurrently
func (e *Executor) executeLevel(steps []Step) error {
        var wg sync.WaitGroup
        errCh := make(chan error, len(steps))

        for _, step := range steps {
                wg.Add(1)
                go func(s Step) {
                        defer wg.Done()
                        if err := e.executeStep(s); err != nil {
                                errCh <- err
                        }
                }(step)
        }

        wg.Wait()
```

```go
}
```

## 5. Files to Modify/Create
- `cmd/run.go`: Add `--parallel` flag to `runCmd` and pass it to the executor.
- `pipeline/executor.go`: Add `Parallel` field to `Executor` struct and `NewExecutor` function. Modify `executeLevel` to honor the `Parallel` flag (sequential if false).

## 6. Constraints and Gotchas
- **Current Behavior**: The `Executor` currently *always* runs steps within a level concurrently in `executeLevel`. The task to "enable" it via a flag implies the default should be changed to sequential.
- **Error Handling**: Use `fmt.Errorf("context: %w", err)` for wrapping errors.
- **IO**: Use `cmd.OutOrStdout()` and `cmd.ErrOrStderr()` in command implementations.
- **Cobra**: Flags should be registered in `init()`.
- **TopologicalSort**: Returns levels where steps within a level are independent.
- **Testing**: Use `t.TempDir()` for all file system operations in tests.
