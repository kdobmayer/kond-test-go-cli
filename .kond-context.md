# Context Bundle: Add --quiet flag to run command

This bundle provides the necessary context for implementing a `--quiet` flag in the `run` command of the `pipeline` CLI tool.

```bash

```

## Relevant Code Snippets

### Flag Registration (cmd/run.go)
Flags for the `run` command are registered in its `init()` function:
```go
func init() {
	rootCmd.AddCommand(runCmd)
	runCmd.Flags().Bool("dry-run", false, "Show execution plan without running")
	runCmd.Flags().Bool("verbose", false, "Show step output in real-time")
}
```

### Retrieving Flag Values (cmd/run.go)
Flag values are retrieved using the `cmd.Flags()` method:
```go
func runPipeline(cmd *cobra.Command, args []string) error {
	pipelineFile := args[0]
	dryRun, _ := cmd.Flags().GetBool("dry-run")
    // ...
}
```

### Output Handling Patterns (cmd/run.go)
Standard output should use `cmd.OutOrStdout()` and error output should use `cmd.ErrOrStderr()`:
```go
fmt.Fprintf(cmd.OutOrStdout(), "Running pipeline %q (run: %s)...\n", p.Name, executor.Run.RunID)

// Rendering complex output via formatter
formatter := output.NewFormatter(outputFormat, cmd.OutOrStdout())
if err := formatter.Render(headers, rows, executor.Run); err != nil {
    return fmt.Errorf("rendering output: %w", err)
}

// Error output
if errs := p.Validate(); len(errs) > 0 {
    fmt.Fprintln(cmd.ErrOrStderr(), "Pipeline validation failed:")
    // ...
}
```

### Error Handling Convention
Errors are wrapped with context:
```go
if err != nil {
    return fmt.Errorf("loading pipeline: %w", err)
}
```

## Test Patterns

Existing tests for the CLI commands are located in `cmd/cmd_test.go`. They follow this pattern:
```go
func TestRunCommand_DryRun(t *testing.T) {
	dir := t.TempDir()
	path := filepath.Join(dir, "pipeline.yaml")
	os.WriteFile(path, []byte(`...`), 0644)

	rootCmd.SetArgs([]string{"run", path, "--dry-run"})
	var buf bytes.Buffer
	rootCmd.SetOut(&buf)

	if err := rootCmd.Execute(); err != nil {
		t.Fatalf("run --dry-run error = %v", err)
	}

	out := buf.String()
	if !bytes.Contains([]byte(out), []byte("Level")) {
		t.Error("expected execution plan output")
	}
}
```

## Files to Modify

## Constraints and Gotchas
- **Silence Usage/Errors**: Note that `main.go` handles printing the final error to `os.Stderr`.
- **Output Consistency**: Always use `cmd.OutOrStdout()` and `cmd.ErrOrStderr()` to ensure tests can capture output.
- **Quiet Definition**: The `--quiet` flag should suppress "non-error" output. This typically includes progress messages, execution plans, and result tables. Validation errors and final execution failures should still be visible to the user.
- **Zero-Config**: `config.Load()` handles missing config files; ensure your changes don't break this.
