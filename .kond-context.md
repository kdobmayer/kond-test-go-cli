# Context Bundle for Implementation Agent

## Goal
Add a `--verbose` flag to the `run` command that prints each step name before executing it.

## Test Command
`go test ./...`

## Build Command
`go build -o pipeline main.go`

## Relevant Code Snippets

### Existing Pattern for Flag Registration (`cmd/run.go`)
```go
func init() {
	rootCmd.AddCommand(runCmd)
	runCmd.Flags().Bool("dry-run", false, "Show execution plan without running")
	runCmd.Flags().Bool("verbose", false, "Show step output in real-time")
}
```

## Test Patterns (`cmd/cmd_test.go`)
Tests use `rootCmd.SetArgs` and `rootCmd.SetOut` to capture output.
```go
func TestRunCommand_DryRun(t *testing.T) {
	// ... (setup pipeline file)

	rootCmd.SetArgs([]string{"run", path, "--dry-run"})
	var buf bytes.Buffer
	rootCmd.SetOut(&buf)

	if err := rootCmd.Execute(); err != nil {
		t.Fatalf("run --dry-run error = %v", err)
	}

	out := buf.String()
	if !bytes.Contains([]byte(out), []byte("Level")) {
		t.Error("expected execution plan output")
	}
}
```

## Files to Modify
- `cmd/run.go`: To retrieve the `--verbose` flag and pass it to the executor or handle the printing.
- `pipeline/executor.go`: To implement the verbose printing of step names before execution.

## Constraints and Gotchas
- **Error Handling**: Use `fmt.Errorf("context: %w", err)` for wrapping errors.
- **IO**: Use `cmd.OutOrStdout()` and `cmd.ErrOrStderr()` for output to ensure testability.
- **Project Structure**: `pipeline/` directory is ignored by `.gitignore`.
- **Parallel Execution**: Steps might be executed in parallel. Ensure verbose output is handled safely.
- **Dependencies**: Prefer standard library over external dependencies.
