#### 3.1. `cmd/root.go` - Global Flag Definition Example
This snippet shows how `PersistentFlags()` are used to define global flags, like the `--output` flag. The `--verbose` flag should follow a similar pattern.

```go

import (
	"github.com/spf13/cobra"
)

	outputFormat string
```

#### 3.2. `config/config.go` - LogLevel Configuration and Validation
This demonstrates the `Config` struct, default values, and how `LogLevel` is managed and validated. The `--verbose` flag should ideally set the `LogLevel` to `"debug"`.

```go
package config

import (
	"fmt"
	"os"
	"path/filepath"
	"strings"

	"gopkg.in/yaml.v3"
)

// Config represents the application configuration
type Config struct {
	PipelineDir string            `yaml:"pipeline_dir"`
	RunDir      string            `yaml:"run_dir"`
	LogLevel    string            `yaml:"log_level"`
	Output      string            `yaml:"output"`
	Defaults    map[string]string `yaml:"defaults,omitempty"`
}

// DefaultConfig returns the default configuration
func DefaultConfig() *Config {
	home, _ := os.UserHomeDir()
	return &Config{
		PipelineDir: filepath.Join(home, ".pipeline", "pipelines"),
		RunDir:      filepath.Join(home, ".pipeline", "runs"),
		LogLevel:    "info",
		Output:      "table",
		Defaults:    make(map[string]string),
	}
}

// Set updates a config value by key
func (c *Config) Set(key, value string) error {
	switch key {
	case "pipeline_dir":
		c.PipelineDir = value
	case "run_dir":
		c.RunDir = value
	case "log_level":
		if !isValidLogLevel(value) {
			return fmt.Errorf("invalid log level: %s (valid: debug, info, warn, error)", value)
		}
		c.LogLevel = value
	case "output":
		if !isValidOutput(value) {
			return fmt.Errorf("invalid output format: %s (valid: table, json, yaml)", value)
		}
		c.Output = value
	default:
		if strings.HasPrefix(key, "defaults.") {
			subKey := strings.TrimPrefix(key, "defaults.")
			if c.Defaults == nil {
				c.Defaults = make(map[string]string)
			}
			c.Defaults[subKey] = value
			return nil
		}
		return fmt.Errorf("unknown config key: %s", key)
	}
	return nil
}

func isValidLogLevel(level string) bool {
	switch level {
	case "debug", "info", "warn", "error":
		return true
	}
	return false
}
```

### 4. Files to Modify/Create

*   `cmd/root.go`:
    *   Add a new global boolean variable for the verbose flag.
    *   Register the `--verbose` persistent flag in the `init()` function using `rootCmd.PersistentFlags().BoolVarP()`.
    *   Add logic to `Execute()` (or a new function called before `rootCmd.Execute()`) to load the config, and if `verbose` is true, update `cfg.LogLevel` to `"debug"`.
*   `cmd/*.go` files (e.g., `cmd/status.go`, `cmd/run.go`):
    *   Where debug output is desired, add conditional checks like `if verboseFlag { fmt.Fprintln(cmd.ErrOrStderr(), "Debug message") }`. This `verboseFlag` would need to be accessible from these files. One way is to export it from `cmd/root.go` or pass it via context if a more sophisticated logging setup is introduced. For simplicity, we can reuse the `config.LogLevel` to control debug output. This means we'd pass the loaded `config.Config` object to functions needing logging capabilities.

### 5. Constraints

*   **Error Handling:** Errors must be consistently wrapped using `fmt.Errorf("context: %w", err)`. Functions return `(Type, error)`, requiring `nil` checks. `cobra.Command` `RunE` functions MUST return an `error`. Only `main.go` calls `os.Exit(1)` after printing the final error to `os.Stderr`.
*   **CLI Output:** All command output (stdout and stderr) MUST use `cmd.OutOrStdout()` and `cmd.ErrOrStderr()` respectively. Direct `fmt.Println` or `os.Stdout`/`os.Stderr` are only for `main.go` or specific `output` package functions. Debug output from `--verbose` should go to `cmd.ErrOrStderr()`.
*   **Logging Integration:** The `--verbose` flag should ideally set the `config.LogLevel` to `"debug"`. Any debug messages should only print when `config.LogLevel` is "debug" or when the `--verbose` flag is explicitly set.
*   **Config Access:** `config.Load()` and `config.Set(key, value)` are available for managing configuration.
