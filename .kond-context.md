### Flag Handling (cmd/run.go)
```go
func runPipeline(cmd *cobra.Command, args []string) error {
	dryRun, _ := cmd.Flags().GetBool("dry-run")
	
	// Load and validate pipeline...
	
	levels, err := pipeline.TopologicalSort(p.Steps)
	if err != nil {
		return fmt.Errorf("computing execution order: %w", err)
	}

	if dryRun {
		fmt.Fprintf(cmd.OutOrStdout(), "Execution plan for %q:\n\n", p.Name)
		for i, level := range levels {
			fmt.Fprintf(cmd.OutOrStdout(), "Level %d (parallel):\n", i+1)
			for _, step := range level {
				fmt.Fprintf(cmd.OutOrStdout(), "  - %s: %s\n", step.Name, step.Command)
			}
		}
		return nil
	}
	// ... rest of execution
}
```

### Test Patterns (cmd/cmd_test.go)
```go
func TestRunCommand_DryRun(t *testing.T) {
	dir := t.TempDir()
	path := filepath.Join(dir, "pipeline.yaml")
	os.WriteFile(path, []byte(`...`), 0644)

	rootCmd.SetArgs([]string{"run", path, "--dry-run"})
	var buf bytes.Buffer
	rootCmd.SetOut(&buf)

	if err := rootCmd.Execute(); err != nil {
		t.Fatalf("run --dry-run error = %v", err)
	}

	out := buf.String()
	if !bytes.Contains([]byte(out), []byte("Level")) {
		t.Error("expected execution plan output")
	}
}
```

## Test Patterns
- Use `t.TempDir()` for isolated test environments.
- Capture output using `bytes.Buffer` and `rootCmd.SetOut`/`rootCmd.SetErr`.
- Simulate CLI arguments using `rootCmd.SetArgs`.
- Assert on both success (no error returned from `Execute`) and expected substrings in output.

## Files to Modify/Create
- `cmd/run.go`: Register flag and implement logic.
- `cmd/cmd_test.go`: Add test case for dry-run functionality.

## Constraints and Gotchas
- **Output Consistency:** Use `cmd.OutOrStdout()` for standard output and `cmd.ErrOrStderr()` for error messages.
- **Error Handling:** Use `fmt.Errorf("context message: %w", err)` to wrap errors.
- **Framework:** The project uses Cobra. Persistent flags like `--output` are registered on `rootCmd`.
- **Pipeline Logic:** Use `pipeline.LoadPipeline(file)` to load and `pipeline.TopologicalSort(steps)` to determine execution order.
- **Intentional Duplication:** Some commands contain duplicated output formatting logic (e.g., table rendering). Follow existing patterns in the file you are modifying.
- **Zero-Config:** Ensure commands handle missing configuration gracefully (e.g., `config.Load()` returns defaults).
