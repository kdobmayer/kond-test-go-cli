# Context Bundle: Add --limit flag to logs command

## Goal
Implement a `--limit` (or `-n`) flag for the `logs` command that caps the output to the last `N` lines of stdout/stderr.

## Commands
- **Test command:** `go test ./...`
- **Build command:** `go build -o pipeline .`

## Files to Modify/Create
- `cmd/logs.go`: Register the `--limit` flag and implement the line-capping logic in the rendering functions.
- `cmd/logs_test.go`: Create a new test file to verify the `--limit` flag behavior.

## Relevant Code Snippets

### Flag Registration (`cmd/logs.go`)
Existing flags are registered in `init()`:
```go
func init() {
	rootCmd.AddCommand(logsCmd)
	logsCmd.Flags().Bool("stderr", false, "Show only stderr output")
	logsCmd.Flags().Bool("stdout", false, "Show only stdout output")
}
```

### Log Data Structure (`pipeline/model.go`)
```go
type StepLog struct {
	StepName string `yaml:"step_name" json:"step_name"`
	Stdout   string `yaml:"stdout" json:"stdout"`
	Stderr   string `yaml:"stderr" json:"stderr"`
}
```

### Log Rendering Pattern (`cmd/logs.go`)
The `renderStepLog` function currently prints the entire log:
```go
func renderStepLog(cmd *cobra.Command, log *pipeline.StepLog, showStdout, showStderr bool) error {
	switch outputFormat {
	case "json":
		data := buildLogOutput(log, showStdout, showStderr)
		// ... json encoding ...
	case "yaml":
		data := buildLogOutput(log, showStdout, showStderr)
		// ... yaml encoding ...
	default:
		fmt.Fprintf(cmd.OutOrStdout(), "=== Step: %s ===\n", log.StepName)
		if !showStderr {
			fmt.Fprintf(cmd.OutOrStdout(), "\n--- stdout ---\n%s", log.Stdout)
		}
		if !showStdout {
			fmt.Fprintf(cmd.OutOrStdout(), "\n--- stderr ---\n%s", log.Stderr)
		}
		fmt.Fprintln(cmd.OutOrStdout())
		return nil
	}
}
```

## Constraints and Gotchas
- **Error Handling:** Wrap errors with context using `fmt.Errorf("context: %w", err)`.
- **IO:** Use `cmd.OutOrStdout()` and `cmd.ErrOrStderr()` instead of `os.Stdout/Stderr`.
- **Output Format:** The global `outputFormat` variable (from `root.go`) determines whether to render as table (default), JSON, or YAML.
- **Log Source:** Logs are loaded via `pipeline.LoadStepLog(runDir, runID, stepName)`.
- **Line Capping:** The limit should apply to both `stdout` and `stderr` independently. If `--limit 10` is provided, show only the last 10 lines of each.
- **Topological Sort:** Pipeline steps are executed in levels. The `logs` command (without a step name) iterates through all steps in a run.
