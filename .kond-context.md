## 4. Test Patterns

### `cmd/cmd_test.go` - Example Test Cases

This snippet illustrates the general testing approach for commands. It shows how to:
- Use `t.TempDir()` for temporary test environments.
- Set command-line arguments with `rootCmd.SetArgs()`.
- Capture `stdout` and `stderr` using `bytes.Buffer` and `rootCmd.SetOut`/`SetErr`.
- Assert on command execution results (error or no error).
- Verify side effects (e.g., file creation) or output content.

```go
package cmd

import (
	"bytes"
	"os"
	"path/filepath"
	"testing"
)

func TestInitCommand(t *testing.T) {
	dir := t.TempDir()

	rootCmd.SetArgs([]string{"init", "test-pipeline", "-d", dir})
	var buf bytes.Buffer
	rootCmd.SetOut(&buf)

	if err := rootCmd.Execute(); err != nil {
		t.Fatalf("init command error = %v", err)
	}

	// Verify file was created
	path := filepath.Join(dir, "test-pipeline.yaml")
	if _, err := os.Stat(path); err != nil {
		t.Errorf("pipeline file not created: %v", err)
	}
}

func TestValidateCommand_Invalid(t *testing.T) {
	dir := t.TempDir()
	path := filepath.Join(dir, "invalid.yaml")
	os.WriteFile(path, []byte(`
name: ""
steps: []
`), 0644)

	rootCmd.SetArgs([]string{"validate", path})
	var buf bytes.Buffer
	rootCmd.SetOut(&buf)
	rootCmd.SetErr(&buf)

	err := rootCmd.Execute()
	if err == nil {
		t.Error("expected error for invalid pipeline")
	}
}
```

## 6. Constraints and Gotchas

-   **Version String Definition:** The application version string (e.g., "1.0.0") is not currently defined in the codebase. It will need to be introduced, likely as a package-level variable in `cmd/root.go` (or `main.go`), which can optionally be populated via `ldflags` during the build process for release versions. For the purpose of adding the flag, a simple constant string will suffice initially.
-   **Cobra `SilenceUsage` / `SilenceErrors`:** The `rootCmd` does not explicitly set `SilenceUsage` or `SilenceErrors`. This means Cobra might print usage information on error or after successful execution of a command. For a `--version` flag, the expected behavior is typically to print the version and exit cleanly without showing the general usage message. This might require setting `rootCmd.SilenceUsage = true` or handling the version output within a custom `Run` function for the flag.
-   **Flag Precedence:** The `--version` flag should ideally short-circuit any other command execution and just print the version, then exit.
-   **Output:** The version output should go to `stdout`.
